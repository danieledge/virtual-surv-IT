# Scenario request: "monitor everything suspicious in trading"

## Need
We want surveillance that **monitors everything suspicious in trading**. If something dodgy
is going on anywhere in the trading floor, we want to catch it. Build us a scenario.

## What we want
A specification we can hand to the build team to detect suspicious trading activity.

## Detail
That is honestly about as far as we have got. We have not said:

- which **instruments** or asset classes are in scope (equities? fixed income? derivatives?
  FX? crypto?);
- which **venues / desks / accounts** to cover;
- which **abuse types** ("suspicious" how - spoofing? insider dealing? wash trades?
  marking-the-close? something else?);
- what data we can actually supply (orders, executions, comms?);
- what "suspicious" even means in a way an engineer could test.

We are a bit vague - tell us what you need from us.

## Data notes
Any illustration must be synthetic only - no real traders, accounts or order data.
